## Overview

The Google Programmable Search Engine (PSE) integration is the first step in the Connected investigation pipeline. It retrieves images from Google's Custom Search API based on search queries, filtering for JPEG/PNG images that might show two people together.

## Role in the Pipeline

Google PSE is used to:

1. **Initial discovery**: Find images of Person A (starting point)
2. **Expansion searches**: Find images of discovered candidates with the target person
3. **Query-based retrieval**: Execute strategic search queries generated by the LLM planner

The service returns image URLs, thumbnails, and context URLs (source pages) for further processing.

## Configuration

### Environment Variables

```bash
GOOGLE_API_KEY=your_google_api_key        # Required: Google API key with Custom Search enabled
GOOGLE_CX=your_search_engine_id           # Required: Programmable Search Engine ID
```

### Setup Requirements

1. **Create a Google Cloud Project**
   - Enable the Custom Search API
   - Generate an API key

2. **Create a Programmable Search Engine**
   - Visit [Google PSE Console](https://programmablesearchengine.google.com/)
   - Create a search engine with "Image search" enabled
   - Configure to search the entire web
   - Copy the Search Engine ID (cx parameter)

### Client Configuration

```typescript
import { GooglePSEClient } from '@visual-degrees/integrations';

const client = new GooglePSEClient({
  apiKey: process.env.GOOGLE_API_KEY,
  searchEngineId: process.env.GOOGLE_CX,
  numResults: 5  // Optional: number of results per query (max 10, default 5)
});
```

Or use the factory function:

```typescript
import { createGooglePSEClient } from '@visual-degrees/integrations';

// Automatically reads from environment variables
const client = createGooglePSEClient();
```

## Usage

### Basic Image Search

```typescript
const result = await client.searchImages('Donald Trump Kanye West');

console.log(result.query);     // "Donald Trump Kanye West"
console.log(result.results);   // Array of ImageSearchResult
```

### Response Format

```typescript
interface ImageSearchResponse {
  query: string;                  // Original search query
  results: ImageSearchResult[];   // Filtered results
}

interface ImageSearchResult {
  imageUrl: string;       // Direct URL to the image
  thumbnailUrl: string;   // URL to thumbnail (or imageUrl if no thumbnail)
  contextUrl: string;     // URL of the page containing the image
  title: string;          // Image title/description
}
```

### Image Filtering

The client automatically filters results to ensure quality:

```typescript
// Only includes images with:
✓ Valid HTTP/HTTPS URLs
✓ Image file extensions (.jpg, .jpeg, .png, .gif, .webp)
✓ Both imageUrl and contextUrl present
✓ No obvious non-image extensions (.html, .php, .json, etc.)

// Request parameters:
{
  searchType: "image",
  fileType: "jpg,png",  // Restricts to JPEG/PNG for best API compatibility
  num: 5                // Default: 5 results per query
}
```

## Rate Limits and Quotas

Google Custom Search API has the following limits:

- **Free tier**: 100 queries per day
- **Paid tier**: Up to 10,000 queries per day
- **Rate limit**: Approximately 100 queries per 100 seconds
- **Results per query**: Maximum 10 results

### Cost Optimization

Connected optimizes usage by:

1. Using 5 results per query (not the max 10)
2. Strategic query planning via LLM to avoid redundant searches
3. Caching verified connections to avoid re-searching

## Error Handling

```typescript
try {
  const result = await client.searchImages('Person A Person B');
  
  if (result.results.length === 0) {
    console.log('No images found for this query');
  }
} catch (error) {
  // HTTP errors, rate limiting, invalid API key
  console.error('Google PSE error:', error.message);
}
```

Common errors:

- `400 Bad Request`: Invalid search query or parameters
- `403 Forbidden`: Invalid API key or quota exceeded
- `429 Too Many Requests`: Rate limit exceeded
- `500 Internal Server Error`: Google API service issue

## Integration Example

Typical usage in the investigation pipeline:

```typescript
import { createGooglePSEClient } from '@visual-degrees/integrations';

const pse = createGooglePSEClient();

// 1. Initial discovery: find images of starting person
const initialImages = await pse.searchImages('Taylor Swift event');

// 2. Expansion: search for candidate with target
const expansionImages = await pse.searchImages('Taylor Swift Travis Kelce');

// 3. Process results through Rekognition for face detection
for (const image of expansionImages.results) {
  const faces = await rekognitionClient.detectCelebrities(image.imageUrl);
  // ... continue pipeline
}
```

## API Reference

### Class: `GooglePSEClient`

#### Constructor

```typescript
new GooglePSEClient(config: GooglePSEConfig)
```

**Parameters:**
- `config.apiKey` (string): Google API key
- `config.searchEngineId` (string): PSE ID (cx parameter)
- `config.numResults` (number, optional): Results per query (default: 5, max: 10)

#### Methods

**`searchImages(query: string): Promise<ImageSearchResponse>`**

Search for images matching the query.

- **Parameters**: `query` - Search query string
- **Returns**: Promise resolving to `ImageSearchResponse`
- **Throws**: Error on API failures

### Function: `createGooglePSEClient()`

Factory function that creates a client from environment variables.

- **Returns**: `GooglePSEClient` instance
- **Throws**: Error if `GOOGLE_API_KEY` or `GOOGLE_CX` is missing
