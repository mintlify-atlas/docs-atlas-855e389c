## Overview

Cloudflare Workers power the Connected API, handling all HTTP requests, WebSocket upgrades, and orchestrating the investigation workflow. The worker serves as the main entry point for all client interactions.

## Configuration

The worker is configured in `wrangler.toml`:

```toml
name = "visual-degrees-worker"
main = "src/index.ts"
compatibility_date = "2024-12-01"
compatibility_flags = ["nodejs_compat"]

[vars]
AWS_REGION = "us-east-1"
ALLOWED_ORIGINS = "https://vd.nintang48.workers.dev,http://localhost:3000,http://localhost:3001"

[ai]
binding = "AI"

[observability]
enabled = true
head_sampling_rate = 1
```

## API Endpoints

The worker exposes the following HTTP endpoints:

### Investigation Endpoints

#### POST `/api/chat/query`
Starts a new investigation between two people.

**Request:**
```json
{
  "personA": "Elon Musk",
  "personB": "Taylor Swift"
}
```

**Response:**
```json
{
  "id": "workflow-instance-id",
  "runId": "uuid",
  "status": "started",
  "personA": "Elon Musk",
  "personB": "Taylor Swift"
}
```

**Rate Limiting:**
- Enforces 50 searches per day per IP
- Returns `429 Too Many Requests` when limit exceeded
- Whitelisted IPs bypass rate limits

**Implementation:**
```typescript
// apps/worker/src/index.ts:199
if (url.pathname === "/api/chat/query" && request.method === "POST") {
  const clientIP = getClientIP(request);
  const rateLimit = await checkRateLimit(env, clientIP);

  if (!rateLimit.allowed) {
    return new Response(JSON.stringify({
      error: "Rate limit exceeded. You can perform 50 searches per day.",
      remaining: 0,
      resetAt: resetDate.toISOString(),
    }), {
      status: 429,
      headers: {
        "X-RateLimit-Limit": String(RATE_LIMIT_MAX),
        "X-RateLimit-Remaining": "0",
        "X-RateLimit-Reset": String(rateLimit.resetAt),
      }
    });
  }

  const instance = await env.INVESTIGATION_WORKFLOW.create({
    params: { personA, personB, runId }
  });
  // ...
}
```

### Real-Time Updates

#### WebSocket `/api/chat/ws/:runId`
Real-time investigation events via WebSocket (powered by Durable Objects).

**Connection:**
```typescript

ws.onmessage = (event) => {
  const message = JSON.parse(event.data);
  // Handle event: message.data contains InvestigationEvent
};
```

**Implementation:**
```typescript
// apps/worker/src/index.ts:142
if (url.pathname.startsWith("/api/chat/ws/") && 
    request.headers.get("Upgrade") === "websocket") {
```

## CORS Configuration

The worker implements dynamic CORS headers based on allowed origins:

```typescript
// apps/worker/src/index.ts:27
function getAllowedOrigins(env: Env): string[] {
  if (env.ALLOWED_ORIGINS) {
    return env.ALLOWED_ORIGINS.split(",").map(o => o.trim());
  }
  return DEFAULT_ALLOWED_ORIGINS;
}

function getCorsHeaders(request: Request, env: Env): Record<string, string> {
  const allowedOrigins = getAllowedOrigins(env);
  const origin = request.headers.get("Origin") || "";
  const allowedOrigin = allowedOrigins.includes(origin) 
    ? origin 
    : allowedOrigins[0];

  return {
    "Access-Control-Allow-Origin": allowedOrigin,
    "Access-Control-Allow-Methods": "GET, POST, OPTIONS",
    "Access-Control-Allow-Headers": "Content-Type",
  };
}
```

## Rate Limiting

Rate limits are enforced at the worker level using KV storage:

```typescript
// apps/worker/src/index.ts:18
const RATE_LIMIT_MAX = 50;
const RATE_LIMIT_WINDOW = 24 * 60 * 60; // 24 hours

async function checkRateLimit(
  env: Env,
  ip: string
): Promise<{ allowed: boolean; remaining: number; resetAt: number }> {
  if (isWhitelisted(env, ip)) {
    return { allowed: true, remaining: 999999, resetAt: 0 };
  }

  const key = `ratelimit:${ip}`;
  const data = await env.RATE_LIMIT.get(key);
  
  if (!data) {
    const resetAt = now + RATE_LIMIT_WINDOW;
    await env.RATE_LIMIT.put(key, JSON.stringify({ count: 1, resetAt }), {
      expirationTtl: RATE_LIMIT_WINDOW,
    });
    return { allowed: true, remaining: RATE_LIMIT_MAX - 1, resetAt };
  }
  // ...
}
```

See [KV documentation](/integrations/kv) for more details on rate limiting storage.

## Client IP Detection

```typescript
// apps/worker/src/index.ts:118
function getClientIP(request: Request): string {
  return request.headers.get("CF-Connecting-IP") ||
         request.headers.get("X-Forwarded-For")?.split(",")[0]?.trim() ||
         "unknown";
}
```

## Architecture Role

The Worker serves as the **orchestration layer** in the Connected architecture:

1. **Request Handler**: Receives and validates all HTTP/WebSocket requests
2. **Rate Limiter**: Enforces usage limits via KV storage
3. **Workflow Dispatcher**: Triggers investigation workflows
4. **Real-time Router**: Routes WebSocket connections to appropriate Durable Objects
5. **Data Access**: Queries D1 database for graph data
6. **CORS Gateway**: Manages cross-origin access
