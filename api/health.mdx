## GET /api/health

Comprehensive health check that tests all external service dependencies.

### Description

This endpoint performs live tests of all third-party integrations required for the Connected service:

1. **Google Custom Search API** - Image search functionality
2. **AWS Rekognition** - Celebrity detection in images
3. **OpenRouter (Gemini)** - AI query parsing and reasoning

### Response

<ResponseField name="status" type="string">
  Overall service health: "healthy" or "degraded"
</ResponseField>

<ResponseField name="services" type="object">
  Detailed health status for each external service
</ResponseField>

<ResponseField name="timestamp" type="string">
  ISO-8601 timestamp of the health check
</ResponseField>

### Service Health Schema

Each service in the `services` object has:

<ResponseField name="ok" type="boolean">
  Whether the service is functioning correctly
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable status message or error description
</ResponseField>

<ResponseField name="latency" type="number" optional>
  Response time in milliseconds (if test succeeded)
</ResponseField>

### Example Request

```bash
curl https://your-worker.workers.dev/api/health
```

## Health Check Details

### Google Custom Search Test

Performs a real image search query:

```
Query: "Elon Musk"
Expected: At least 1 result returned
```

**Common Failures:**
- Invalid API key (`GOOGLE_API_KEY`)
- Invalid Search Engine ID (`GOOGLE_SEARCH_ENGINE_ID`)
- API quota exceeded
- Network connectivity issues

### AWS Rekognition Test

Analyzes a public celebrity image:

```
Test Image: Elon Musk (Wikipedia)
Expected: At least 1 celebrity detected
```

**Common Failures:**
- Invalid AWS credentials (`AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`)
- Incorrect region (`AWS_REGION`)
- IAM permissions missing for Rekognition
- Service quota exceeded

### OpenRouter Test

Parses a sample query using Gemini:

```
Query: "Elon Musk to Beyonce"
Expected: Valid parse with personA and personB extracted
```

**Common Failures:**
- Invalid API key (`OPENROUTER_API_KEY`)
- Model unavailable or deprecated
- Rate limits or quota exceeded
- Network connectivity issues

## Monitoring Recommendations

### Uptime Monitoring

Set up automated health checks:

```bash
# Check every 5 minutes
*/5 * * * * curl -f https://your-worker.workers.dev/api/health || alert
```

### Success Criteria

- Status 200 (not 503)
- `status` field equals `"healthy"`
- All services have `ok: true`

### Alert Thresholds

**Critical:**
- Multiple services failing simultaneously
- Persistent failures for >15 minutes

**Warning:**
- Single service failure
- Latency >5000ms on any service
- Any service failure lasting >5 minutes

### Latency Benchmarks

Typical response times:

| Service | Typical Latency | Slow Threshold |
|---------|----------------|----------------|
| Google Search | 200-500ms | >2000ms |
| AWS Rekognition | 400-800ms | >3000ms |
| OpenRouter | 800-1500ms | >5000ms |
| **Total Health Check** | **1.5-3s** | **>10s** |

## Environment Variables

Required environment variables for health checks:

```bash
# Google Custom Search
GOOGLE_API_KEY="your-api-key"
GOOGLE_SEARCH_ENGINE_ID="your-search-engine-id"

# AWS Rekognition
AWS_REGION="us-east-1"
AWS_ACCESS_KEY_ID="your-access-key"
AWS_SECRET_ACCESS_KEY="your-secret-key"

# OpenRouter
OPENROUTER_API_KEY="your-api-key"
```

## Troubleshooting

### All Services Failing

1. Check Worker deployment status in Cloudflare dashboard
2. Verify environment variables are set correctly
3. Check Cloudflare Workers logs for startup errors
4. Verify network connectivity from Workers to external APIs

### Google Search Failing

```json
{
  "ok": false,
  "message": "Request failed with status code 403"
}
```

**Solutions:**
- Verify `GOOGLE_API_KEY` is valid and active
- Check API is enabled in Google Cloud Console
- Verify daily quota hasn't been exceeded
- Ensure Search Engine ID is correct

### AWS Rekognition Failing

```json
{
  "ok": false,
  "message": "InvalidSignatureException"
}
```

**Solutions:**
- Verify AWS credentials are correct
- Check IAM user has `rekognition:RecognizeCelebrities` permission
- Ensure region is supported (e.g., us-east-1)
- Check for clock skew on Workers (AWS signature validation)

### OpenRouter Failing

```json
{
  "ok": false,
  "message": "Unauthorized"
}
```

**Solutions:**
- Verify `OPENROUTER_API_KEY` is valid
- Check API key has sufficient credits
- Ensure model `google/gemini-2.0-flash-001` is available
- Check for rate limiting or quota issues
