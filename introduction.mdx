# Introduction to Connected

Connected is a chat-first investigation system that finds **visual degrees of separation** between public figures using live image search, celebrity face verification, and AI-guided reasoning.

## What It Does

Given two public figures (e.g., "Elon Musk" and "Beyonce"), Connected returns a **verified path** with:

- **Evidence-backed connections** - Images where both people appear together
- **Confidence scores** - Based on face recognition with ≥80% threshold
- **Real-time streaming progress** - Live updates via chat interface
- **Interactive graph visualization** - Visual representation of the connection network

Every connection is backed by **visual evidence** - no hallucinated relationships.

## How It Works

Connected uses a multi-phase investigation pipeline:

<Steps>
  <Step title="Research Phase">
    Suggests potential bridge figures who might connect the two targets
  </Step>
  
  <Step title="Discovery Phase">
    Searches for images of candidate figures using Google Programmable Search Engine
  </Step>
  
  <Step title="Selection Phase">
    Ranks candidates strategically based on likelihood of connection
  </Step>
  
  <Step title="Verification Phase">
    Uses AWS Rekognition to verify faces in images and confirm connections
  </Step>
  
  <Step title="Expansion Phase">
    Extends the path and backtracks if needed until a connection is found
  </Step>
</Steps>

<Note>
The system enforces a maximum of **6 degrees of separation** and requires **≥80% confidence** for face detection.
</Note>

## Key Features

### Visual Evidence Verification

Every connection must be backed by photographic evidence where both people appear in the same image. The system:

1. Fetches images using Google Programmable Search Engine
2. Filters out collages and montages using Gemini Flash vision model
3. Runs AWS Rekognition celebrity detection on real photos
4. Validates that both targets are detected with ≥80% confidence

### Real-Time Streaming

The investigation progress is streamed in real-time to the chat interface using Cloudflare Durable Objects, providing:

- Live status updates
- Discovery progress
- Evidence found notifications
- Final path visualization

### Confidence Scoring

Connections are scored based on facial recognition confidence:

```
Per-Image Score = min(confidence_A, confidence_B)
Edge Confidence = max(per_image_score across all images)
Path Confidence = min(edge_confidence along path)
```

### Interactive Graph Visualization

Built with Sigma.js and Graphology, the graph visualization shows:

- Nodes representing public figures
- Edges representing verified connections
- Force-directed layout for optimal viewing
- Clickable nodes to view evidence images

## Architecture

Connected is built on the Cloudflare developer platform for a serverless, globally distributed architecture:

| Service | Purpose |
|---------|----------|
| **Cloudflare Workers** | API orchestration & business logic |
| **Cloudflare Workflows** | Investigation state machine with durable execution |
| **Cloudflare Durable Objects** | Real-time WebSocket streaming |
| **Cloudflare KV** | Session state & rate limiting |
| **Cloudflare Pages** | Next.js frontend hosting |

<Note>
Cloudflare's edge network provides sub-50ms cold starts and runs code closest to users in 300+ locations worldwide.
</Note>

## Key Constraints

<Warning>
Connected operates under strict constraints to ensure accuracy:

- **Maximum 6 hops** (degrees of separation)
- **Confidence threshold ≥80%** for face detection
- **Visual proof required** for every connection
- **No preprocessing** - Graph is built on-demand per query
</Warning>
